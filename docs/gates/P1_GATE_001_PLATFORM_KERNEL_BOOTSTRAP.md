## Layer

L0-L2:

- L0 Host and Hardware.
- L1 Platform Services.
- L2 Engine Kernel.

## Owns

This gate owns the first implementation slice for:

- build/test skeleton;
- headless host lifecycle;
- platform timer/frame clock;
- logging sink boundary;
- error boundary for startup and shutdown failures;
- deterministic shutdown;
- minimal module lifecycle;
- minimal service registry;
- fast CTest gate command.

## Does Not Own

This gate does not own:

- rendering, RHI, swapchain, shaders, material, capture, or frame oracle;
- resource package parsing, VFS, asset handles, upload scheduling, or original resource meaning;
- audio backend, mixer, voices, BGM, or SE state;
- input action mapping, title menu input, focus routing, or UI input;
- UI framework;
- script VM or native bridge;
- world, scene, actor, component, animation, physics, or gameplay;
- TouhouNewWorld adapter behavior;
- tools/editor/import/export;
- diagnostics report JSON schemas as runtime API.

## Lifecycle

First slice lifecycle:

1. Process enters the headless host.
2. Host constructs platform services.
3. Host initializes logging sink in default and disabled modes.
4. Host creates an engine kernel instance.
5. Kernel registers a fixed test module graph.
6. Kernel starts modules in dependency order.
7. Host ticks the kernel for a fixed number of frames.
8. Kernel shuts modules down in deterministic reverse order.
9. Host releases services and exits with explicit status.

Failure behavior:

- startup failure returns explicit error;
- missing required service returns explicit error;
- module startup failure triggers deterministic teardown of already-started modules;
- shutdown failure is surfaced, not silently ignored.

## Inputs

- command-line arguments required only for headless test configuration;
- fixed test module graph;
- platform clock source;
- logging configuration;
- optional test fixture configuration generated by tests.

## Outputs

- process exit status;
- in-memory lifecycle trace consumed by tests;
- logging events through the logging sink;
- deterministic performance counters for the fixture;
- no JSON report output as runtime API.

## Dependencies

Allowed dependencies:

- C++ standard library;
- OS/platform APIs behind platform boundary;
- CMake/CTest tooling;
- future minimal test framework chosen by implementation only if ADR-0001 accepts it.

Forbidden dependencies:

- TouhouNewWorld `bin` or `resource`;
- old `YuEngine_BACKUP` runtime files;
- old `FrameRuntime.*`;
- renderer/audio/input/resource/script/world modules;
- report dashboards;
- editor/tool code.

## Performance Constraints

Required deterministic signals:

- startup allocation count or a placeholder measurement hook if allocator accounting is not yet implemented;
- peak/retained bytes hook or explicit deferred rationale if allocator work is assigned to P1-GATE-002;
- disabled logging fixture proving logs can be disabled without behavior change;
- module lifecycle fixture records startup/update/shutdown order and count;
- service registry fixture declares lookup behavior and whether hot-path callers must cache handles;
- host tick fixture records fixed tick count and monotonic timer behavior.

Blocking conditions:

- global mutable business state;
- logging/report output required for behavior;
- stringly service lookup treated as hot-path-safe without declared caching rule;
- unbounded task queue or background thread introduced by this slice;
- any dependency on original-game evidence.

## Tests

Fast gate tests required before the slice can be considered complete:

- `Host_StartTickShutdown_Deterministic`
- `Host_TimerMonotonic_ForFixedTicks`
- `Logging_DisabledSink_DoesNotChangeBehavior`
- `Kernel_ModuleLifecycle_DependencyOrder`
- `Kernel_ModuleStartupFailure_TearsDownStartedModules`
- `Kernel_ServiceRegistry_ResolveAndMissingService`

Test command family expected from ADR-0001:

```text
cmake -S . -B build
cmake --build build
ctest --test-dir build --output-on-failure
```

The implementation task may refine build configuration flags, but must preserve a non-interactive fast gate.

## Allowed First Slice

If approved, the first implementation slice may create:

- root CMake project files;
- `src` and `tests` layout required for this slice;
- headless host executable or test harness;
- platform timer/log/error boundary interfaces and minimal implementation;
- kernel module lifecycle and service registry minimal implementation;
- tests listed above.

It may not create placeholder renderer, UI, resource viewer, game loop business, Touhou adapter, or old-runtime migration code.

## Non-Goals

- No game simulation.
- No original resource loading.
- No frame capture/oracle.
- No editor.
- No packaging/import/export.

- No real RHI/audio/input backend.
